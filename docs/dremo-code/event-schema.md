# Proposed Dremo Event Schema

Status: proposed.

Dremo events are ordered, server-owned records. The frontend renders them but must never create trusted runtime events.

## Canonical Structure

```json
{
  "id": "evt_123",
  "taskId": "task_123",
  "sequence": 1,
  "type": "task_created",
  "channel": "system",
  "severity": "info",
  "payload": {},
  "createdAt": "2026-04-28T00:00:00Z"
}
```

## Fields

| Field | Type | Requirement |
| --- | --- | --- |
| `id` | string | Unique event id generated by backend. |
| `taskId` | string | Parent Dremo task id. |
| `sequence` | number | Strictly increasing per task. |
| `type` | string | Event type from the allowed set. |
| `channel` | enum | `system`, `agent`, `terminal`, `tool`, `approval`, `artifact`, `billing`. |
| `severity` | enum | `debug`, `info`, `warning`, `error`. |
| `payload` | object | Type-specific payload. |
| `createdAt` | string | Server timestamp. |

## Channel Guidance

| Channel | Intended content |
| --- | --- |
| `system` | Task lifecycle, policy decisions, orchestration changes. |
| `agent` | Planning, reasoning summaries, self-review summaries, final report milestones. |
| `terminal` | Command output from sandboxed execution. |
| `tool` | Tool calls, tool output, and tool completion. |
| `approval` | User approval requests and resolutions. |
| `artifact` | Files, patches, reports, screenshots, and downloads. |
| `billing` | Credit quote, reservation, charge, release, refund, or review state. |

## Ordering Rules

| Rule | Requirement |
| --- | --- |
| Per-task sequence | `sequence` must be unique and increasing for each task. |
| Append-only | Existing events should not be edited except for internal redaction workflows. |
| Redaction | If sensitive data appears, append a redaction event and mark artifact/log storage accordingly. |
| Replayable UI | Workspace UI should be rebuildable from task summary plus ordered events. |

## Privacy and Safety Rules

| Risk | Requirement |
| --- | --- |
| Secrets in terminal output | Redact before persistence where possible; otherwise quarantine and restrict display. |
| Large output | Truncate stored event payloads and attach full logs as controlled artifacts. |
| Prompt injection | Treat repo files, terminal output, and tool output as untrusted content. |
| User approvals | Store exact action text shown to user so the audit trail matches consent. |
